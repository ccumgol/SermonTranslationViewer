# 설치 및 운영 가이드

## 1. 사전 준비물

| 항목 | 설명 |
|---|---|
| 컴퓨터 | macOS / Windows / Linux (Python 3.10+) |
| 오디오 인터페이스 | 믹서의 AUX/REC OUT → USB 오디오 인터페이스 |
| 유선 인터넷 | 예배 중 안정성을 위해 Wi-Fi 보다 유선 LAN 권장 |
| Gemini API 키 | https://aistudio.google.com/apikey 에서 발급 |

> **권장**: 믹서에서 **목사님 마이크(설교 채널)만 분리된 AUX 버스**를 받으세요.
> 회중석 마이크·반주가 섞인 메인 출력은 인식률이 떨어집니다.

## 2. 설치

```bash
python3 -m venv .venv
source .venv/bin/activate          # Windows: .venv\Scripts\activate
pip install -r requirements.txt
```

`sounddevice` 는 PortAudio 가 필요합니다.
- macOS: `brew install portaudio`
- Ubuntu: `sudo apt install libportaudio2`

## 3. API 키 설정 (민감정보)

### 3-1. 어떤 계정으로 발급해야 하는가 — 중요

**반드시 "개인 Google 계정"(일반 Gmail)으로 발급하세요. 조직(Workspace) 계정은 안 됩니다.**

이 프로젝트가 쓰는 모델 `gemini-3.5-live-translate-preview` 는
**Gemini Developer API(= AI Studio, API 키 방식) 전용**입니다.
아래 두 제약 때문에 인증 방식이 사실상 "개인 계정 + API 키" 하나로 고정됩니다.

| 시도한 방식 | 결과 | 원인 |
|---|---|---|
| 조직 계정에서 API 키 발급 | ❌ 실패 | 조직 보안정책이 API 키 발급을 금지("API 키가 허용되지 않음. ADC를 사용하세요") |
| 조직 계정 + ADC/Vertex AI | ❌ 불가 | `gemini-3.5-live-translate-preview` 가 **Vertex AI 에는 아직 없음**(현재 API 키 전용). [공식 포럼 확인](https://discuss.ai.google.dev/t/pls-realtime-translation-gemini-3-5-live-translate-preview-for-vertex/170660) |
| 개인 계정 + API 키 | ✅ 채택 | 전용 번역 모델이 그대로 동작, 코드 수정 불필요 |

> 정리: **조직 계정으로는 이 전용 번역 모델을 쓸 수 없습니다.**
> Vertex 경로로 가려면 범용 모델(`gemini-2.5-flash` Native Audio)로 갈아타야 하는데,
> 그건 전용 번역 모델이 아니라 설교용 정확도·지연 면에서 불리합니다.
> 그래서 **개인 계정 + API 키**로 확정했습니다.

### 3-2. 결제(Billing) 연결 — Live Translate 필수

`gemini-3.5-live-translate-preview` 는 무료 티어로 동작하지 않습니다.
개인 계정에서 다음을 확인하세요.

1. <https://aistudio.google.com/apikey> 접속 (개인 계정으로 로그인)
2. 좌측 **"결제"** → 사용할 프로젝트가 **"Tier 1 - 유료"** 상태인지 확인
3. 무료 상태면 **"결제 설정"** 버튼으로 결제수단(카드)을 연결
   - ⚠️ 카드번호 입력 등 결제수단 등록은 보안상 **본인이 직접** 진행해야 합니다.

### 3-3. API 키 발급

1. 좌측 **"API 키"** → 우측 상단 **"API 키 만들기"**
2. **키 이름**: `Sermon Live Translate` (구분하기 쉽게)
3. **가져온 프로젝트 선택**: 결제가 연결된 프로젝트 선택
4. **"키 만들기"** → 표시된 `AIza...` 키 복사

> 참고: 자동화(브라우저 봇)로 **새 GCP 프로젝트를 만들면** Google 이
> "의심스러운 요청"으로 차단합니다("The request is suspicious").
> 새 프로젝트 생성은 **사람이 직접** 하세요. 키 발급 자체는 무방합니다.

### 3-4. `.env` 에 저장

```bash
cp .env.example .env
```

`.env` 를 열어 `GEMINI_API_KEY` 에 발급받은 키를 붙여넣습니다.
**이 파일은 절대 외부에 공유하거나 git 에 올리지 마세요.** (`.gitignore` 로 자동 제외됨)

> API 키는 민감정보입니다. 채팅·메신저·캡처 등으로 키 전체를 노출했다면
> AI Studio 에서 해당 키를 삭제하고 **재발급**하세요.

## 4. 오디오 입력 점검

```bash
# 연결된 입력 장치 목록
python server/audio_input.py --list

# 목사님이 마이크에 말할 때 막대가 움직이는지 확인
python server/audio_input.py --monitor --device "USB Audio CODEC"
```

확인한 장치 이름(또는 번호)을 `.env` 의 `AUDIO_INPUT_DEVICE` 에 입력합니다.

## 5. 실행

```bash
python server/ws_server.py
```

| 화면 | 주소 | 용도 |
|---|---|---|
| 송출 자막 | http://localhost:8000/ | 프로젝터 풀스크린 / OBS 브라우저 소스 |
| 운영자 | http://localhost:8000/operator | 한/영 전사·연결상태 모니터링 |

### OBS 연동
1. 소스 추가 → **브라우저**
2. URL: `http://localhost:8000/`
3. 너비/높이를 송출 해상도에 맞춤 (배경 투명이라 영상 위에 자막만 얹힘)

## 6. 운영자 대시보드 사용법

예배 중 모든 조작은 **운영자 화면(`/operator`)** 에서 한다. 터미널이나 `.env` 를
다시 열 필요가 없다.

### 6-1. 오디오 입력 소스 전환
- "오디오 입력 소스" 카드의 드롭다운에서 장치 선택 → **즉시 전환**.
- **번역 세션은 끊기지 않는다**(입력 스트림만 교체). 예: 설교 중 무선 마이크가
  끊기면 `MacBook Pro 마이크` 로 한 번에 전환 가능.
- USB 인터페이스를 예배 중 새로 연결했다면 **🔄 목록 새로고침** 으로 갱신.

### 6-2. 송출 제어
| 버튼 | 동작 |
|---|---|
| **↺ 화면 리셋** | 누적된 자막을 즉시 비움 (설교 단락 전환 시 깔끔하게) |
| **⏹ 송출 종료** | 송출 화면만 비움. 번역은 계속 동작하고 운영자 화면엔 계속 표시됨 |
| **▶ 송출 재개** | 다시 송출 화면에 자막 표시 |

### 6-3. 자막 스타일 조절
- 폰트 크기 / 글자색 / 배경색 / 배경 투명도를 슬라이더·색상 선택으로 조절.
- 변경은 송출 화면에 **실시간 반영**된다. 화면 밝기·배경 영상에 맞춰 가독성 조정.

### 6-4. 번역 언어 (1~3개 동시)
- "번역 언어" 카드의 슬롯 3개로 목표 언어를 1~3개 선택, 각 언어 **글자색** 지정.
- 송출 화면이 언어 수만큼 행으로 나뉘어 각 색으로 표시된다.
- ⚠️ 언어 N개 = 비용/연산 N배.

### 6-5. 백엔드 토글 (온라인 ↔ 오프라인)
- 운영자 화면 상단 **백엔드 배지를 클릭**하면 온라인(Gemini)↔오프라인(로컬) 전환.
- 언어 구성은 유지된다. 오프라인 전환 시 모델 로딩에 ~10초.
- 활용: 평소 온라인, **인터넷 끊기면 클릭 한 번으로 오프라인 폴백**.

> 송출 화면을 새로 열거나 OBS 를 재시작해도, 현재 스타일·언어·송출상태가
> 자동으로 동기화된다.

## 6.5 오프라인(로컬) 모드 설치 — 선택

인터넷·API 비용 없이 동작하는 로컬 백엔드. **Apple Silicon Mac(M1 이상)** 필요.

```bash
# 1) 로컬 STT 패키지
pip install -r requirements-offline.txt        # mlx-qwen3-asr

# 2) 시스템 도구 (이미 있으면 생략)
brew install ollama ffmpeg

# 3) 번역 모델 (~8GB 다운로드)
ollama pull translategemma:12b

# 4) 오프라인으로 실행
BACKEND=local ./start.sh
```

구성: **Qwen3-ASR**(한국어 STT, 침묵으로 발화 구간을 잘라 전사) + **TranslateGemma**
(Ollama 로 번역). 한 번 전사한 한국어를 여러 언어로 번역한다.

튜닝(`.env`, 선택):
- `STT_SILENCE_RMS` — 침묵 판단 음량 임계값. 말해도 자막이 안 뜨면 낮추고(0.008),
  잡음에 반응하면 높인다(0.025).
- `STT_MIN_SILENCE_SEC` — 발화를 끊는 침묵 길이(기본 0.7초).
- `MT_MODEL` — `translategemma:12b`(정확) / `translategemma:4b`(빠름).
- `STT_MODEL` — `Qwen/Qwen3-ASR-0.6B`(빠름·권장) / `Qwen/Qwen3-ASR-1.7B`.

운영 팁: 로컬은 **또렷한 단문 + 문장 끝 짧은 멈춤**일 때 가장 정확하다. 복문·더듬거림은
번역 품질을 떨어뜨리므로 화자에게 짧게 끊어 말하도록 안내하면 좋다.

## 7. 예배 전 체크리스트

- [ ] 믹서 AUX → 인터페이스 → 컴퓨터 입력 레벨 정상 (`--monitor`)
- [ ] 무선 마이크 배터리 확인
- [ ] 유선 인터넷 연결 확인
- [ ] 서버 실행 후 운영자 화면에 "연결됨" 표시
- [ ] 시험 발화 시 1~3초 내 영어 자막 갱신
- [ ] **15분 이상 연속 테스트** → 세션 재개가 끊김 없이 동작하는지 확인

## 8. 장애 시 대응

| 증상 | 조치 |
|---|---|
| 자막이 안 뜸 | 운영자 화면 연결상태 확인 → 서버 콘솔 로그 확인 |
| 자막이 멈춤 | 서버가 자동 재연결(2초). 지속되면 서버 재시작 |
| 소스가 잘못됨 | 운영자 화면에서 **오디오 입력 소스** 드롭다운으로 전환 (세션 안 끊김) |
| 자막 오역 심함 | 입력 음원 품질 점검(다른 소리 섞임?) |
| 자막이 산만함 | 운영자 화면에서 **↺ 화면 리셋** |
| **인터넷 끊김** | 백엔드 배지 클릭 → **오프라인으로 폴백** (로컬 모드 설치돼 있으면) |
| (로컬) 자막 안 뜸 | 입력 장치 확인 + `STT_SILENCE_RMS` 낮추기(0.008) |
| (로컬) 엉뚱한 말 반복 | 입력 장치/채널 확인, 화자가 또렷이 단문으로 |
| 최후 수단 | **⏹ 송출 종료** 로 자막만 끄고 음성 송출은 계속 — 예배 진행에 지장 없음 |

## 9. 비용

- **온라인(Gemini)**: 약 **$0.037/분** × 언어 수. 40분 설교 1언어 ≈ $1.5 미만,
  3언어 ≈ $4.5. 실제 청구는 [AI Studio 결제 페이지](https://ai.google.dev/gemini-api/docs/billing).
- **오프라인(로컬)**: **비용 0** (전기·하드웨어만). 인터넷 불필요.

더 자세한 트러블슈팅은 [troubleshooting-log.md](troubleshooting-log.md) 참고.
